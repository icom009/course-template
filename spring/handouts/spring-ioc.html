IoC — 제어 역전

Spring 기초
SMHRD

학습 목표

  • IoC 의 의미를 한 문장으로 말할 수 있다
  • 제어권이 개발자에서 컨테이너로 넘어가는 발상을 안다
  • Bean 이라는 개념의 큰 그림을 안다

1. IoC = 제어 역전

객체 생성·관리 권한이 개발자가 아닌 외부 컨테이너에게 있는 설계 원칙. 「제어가 뒤집혔다」는 뜻.

비유: 택시 뒷좌석. 운전(객체 생성·관리)은 택시(컨테이너)가, 나는 목적지만 말한다.

2. 코드의 변화

BEFORE
class OrderService {
  MailSender m =
    new GmailSender();
}
AFTER
class OrderService {
  MailSender m;
  // 외부에서 받음
}

3. 누가 만들어주는가

4. 효과

항목변화
결합도↓ 인터페이스에만 의존
유연성↑ 부품 교체 시 클래스 수정 없음
테스트 용이성↑ Mock 으로 쉽게 갈아끼움
코드 간결화new 가 사라짐

학습 확인 체크리스트

  • IoC 를 한 문장으로 설명할 수 있다
  • 택시 뒷좌석 비유를 자기 말로 풀 수 있다
  • Bean 이 「Spring 이 관리하는 인스턴스」라는 사실을 안다
